# Design decisions


## [docker-compose.yml](docker-compose.yml)

Celery beat находится в отдельном контейнере, чтобы не плодить его при 
увеличении реплик Celery worker'a и для простого отключения (контейнера),
если периодические задания станут не нужны.

Миграции отрабатывают в отдельном контейнере, а не в Dockerfile, чтобы
выполняться один раз при запуске (а не в контейнере fastapi, celery и celery beat).

Переменные окружения для запуска всего через докер находятся в docker/.env и
подгружаются докером в env контейнеров.
